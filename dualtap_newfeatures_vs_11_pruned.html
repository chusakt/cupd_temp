0. ที่มา & ภาพรวมการพัฒนาโมเดล

เอกสารนี้สรุปการทบทวนและปรับปรุงโมเดล ML ของงาน dualtap (คัดกรองพาร์กินสัน) ตั้งแต่โมเดลที่ใช้งานจริง (deployed) ไปจนถึงโมเดลที่เราเลือกใช้สุดท้าย. ข้อมูล: preselectPD + preselectControl รวม 1,406 ไฟล์ / 1,189 ผู้ป่วย (PD 622 / Control 784). ทุกตัวเลขผ่าน 10-fold StratifiedGroupKFold (seed 42, group = ผู้ป่วย) → ไม่มี patient leak, รายงานแบบ pooled out-of-fold.

6. ข้อสรุป

ฟีเจอร์ใหม่ + pruning ได้ผลจริง: AUC 0.780 → 0.791, Acc 0.721 → 0.730 โดยใช้ 13 ฟีเจอร์ (เลือกจาก 23). ตัวขับเคลื่อนหลักคือ ความไม่สม่ำเสมอของจังหวะ (tDiff_std, tDiff_cv) — ฟีเจอร์ใหม่ที่สำคัญที่สุด. ยืนยันบทเรียนเดิมว่าการลงทุนกับฟีเจอร์ให้ผลมากกว่าการเปลี่ยนอัลกอริทึม.
เส้นทางสะสมทั้งหมด: original 0.757 → fix features 0.772 → tune RF 0.780 → +new features (pruned 13) 0.791. ขั้นถัดไปที่ยังเหลือ: รวม repeat features (เคยช่วย), patient-level aggregation (#3), และ multimodal fusion (#6) ซึ่งคาดว่าจะดันได้มากกว่า dualtap เดี่ยว.

SHAP ยืนยัน: ความแปรปรวนของจังหวะ (tDiff_std, tDiff_cv) คือสัญญาณ PD ที่แรงและชัดที่สุดทั้งขนาดและทิศทาง — ตรงกับ bradykinesia/arrhythmia. ลายเซ็น PD = จังหวะแปรปรวน + แตะไม่แม่น/ไม่สม่ำเสมอ; ลายเซ็น Control = คะแนนสูง + สลับซ้าย-ขวาสมดุล + มีจังหวะเร็วชัด. 9. หาจุดทำงานที่ Recall สูง — สำหรับคัดกรอง PD

งานคัดกรอง ต้องการ Sensitivity (Recall ของ PD) สูง — พลาดผู้ป่วยให้น้อยที่สุด (False Negative ต่ำ) ยอมแลกกับ False Positive ที่มากขึ้น (ส่งต่อไปตรวจยืนยัน). ตัวชี้วัดสำคัญคือ NPV (ถ้าผลว่า "ปกติ" จะปกติจริงแค่ไหน). ใช้โมเดล RF tuned + 13 ฟีเจอร์ · OOF (group CV) · AUC 0.791 · PD 622 / Control 784 ราย.

10.0 ตัวแปรกลางที่ใช้ร่วม (intermediate variables)

สัญลักษณ์นิยาม
Nจำนวน stroke (การแตะ) = len(taps)
strokeᵢการแตะครั้งที่ i มีจุดสัมผัสหลายจุด แต่ละจุด = {x, y, ts} (ts = epoch ms)
nᵢจำนวนจุดใน strokeᵢ = len(strokeᵢ.data)
(mxᵢ, myᵢ)centroid ของ strokeᵢ = (mean(x), mean(y)) ของจุดในแตะนั้น
tsᵢเวลากลางของ strokeᵢ = mean(ts ของทุกจุดใน strokeᵢ)
tDiffลิสต์ช่วงเวลาระหว่างแตะที่ติดกัน = [ts₂−ts₁, ts₃−ts₂, …] (มี N−1 ค่า, หน่วย ms)
rรัศมีวงกลมเป้า = circleL.r (= 56 ในชุดนี้)
side(จุด)กฎ midpoint รายจุด: ถ้า (x−cLx) ≥ (cRx−x) → ฝั่ง ขวา ไม่งั้น ซ้าย
dist(จุด)ระยะจากจุดถึง "ศูนย์กลางวงกลมฝั่งที่ถูกเลือก" = √((x−cx)²+(y−cy)²)
nL, nRจำนวน stroke ที่ centroid ใกล้วงซ้าย / วงขวา (นับ "ต่อ stroke")

ฟีเจอร์สูตรวิธีคิด
mean_accmean( dist / r ) ทุกจุดระยะถึงเป้าเฉลี่ย หารด้วยรัศมี (normalize ขนาดวงกลม) — ค่า <1 = แตะในวง, ยิ่งสูง = ยิ่งห่างเป้า (แม่นยำต่ำ) → PD

10.4 กลุ่ม D — จังหวะ/เวลา (จาก ts, tDiff)

ฟีเจอร์สูตรวิธีคิด

tDiff_medianmedian(tDiff)ช่วงเวลาค่ากลาง (ทนต่อ outlier กว่า mean)
tDiff_minmin(tDiff)จังหวะที่เร็วที่สุดทำได้ — Control มักมีช่วงเร็วชัด
tDiff_maxmax(tDiff)ช่วงหยุด/ลังเลที่นานที่สุด (freezing/hesitation) → PD
tDiff_stdstd(tDiff)ความไม่สม่ำเสมอของจังหวะ — เคาะไม่เป็นจังหวะคงที่ = arrhythmia ของ PD (ฟีเจอร์ที่ SHAP จัดอันดับ 1)
tDiff_cvstd(tDiff) / mean(tDiff)ความแปรปรวนจังหวะแบบ normalize — แยก "จังหวะรวน" ออกจาก "แค่ช้า" (คนช้าแต่สม่ำเสมอ cv ต่ำ; PD รวน cv สูง)
tDiff_slopeความชัน a จาก fit เส้นตรง tDiff ≈ a·(ลำดับ) + b (least-squares บนดัชนี 0,1,2,…)แนวโน้มว่าช่วงเวลาระหว่างแตะ "เพิ่มขึ้นเรื่อยๆ" ไหมตลอดการทดสอบ. a > 0 = แตะช้าลงเรื่อยๆ (progressive slowing / decrement — ลักษณะเด่นของ PD bradykinesia เวลาเคาะซ้ำๆ); a ≈ 0 = คงที่. ต้องมี ≥3 ค่าถึงคำนวณ ไม่งั้น = 0
duration_s(max(ts) − min(ts)) / 1000เวลารวมที่ใช้ทำงานทั้งหมด (วินาที)
tap_rateN / duration_sอัตราการแตะ (แตะ/วินาที) — ต่ำ = bradykinesia
fatigue_ratiomean(tDiff ครึ่งหลัง) / mean(tDiff ครึ่งแรก)เทียบจังหวะครึ่งหลังกับครึ่งแรก — >1 = ช้าลงเมื่อเวลาผ่านไป (อาการล้า). ต้องมี tDiff ≥4 ค่า

0. Background & model-development overview

This document summarizes the review and improvement of the ML model for the dualtap task (Parkinson's screening), from the deployed model to the final model we chose. Data: preselectPD + preselectControl, 1,406 files / 1,189 patients (PD 622 / Control 784). 6. Conclusion

New features + pruning genuinely help: AUC 0.780 → 0.791, Acc 0.721 → 0.730 using 13 features (selected from 23). The main driver is rhythm irregularity (tDiff_std, tDiff_cv) — the most important new features. This confirms the prior lesson that investing in features beats swapping algorithms.

10. Appendix — rationale + detailed computation of all 23 features

10.0 Shared intermediate variables

Every feature is built from these variables (extracted from recording.taps + circleL/R):

SymbolDefinition
NNumber of strokes (taps) = len(taps)
strokeᵢThe i-th tap, with several touch points, each = {x, y, ts} (ts = epoch ms)
nᵢNumber of points in strokeᵢ = len(strokeᵢ.data)
(mxᵢ, myᵢ)Centroid of strokeᵢ = (mean(x), mean(y)) of its points
tsᵢMid-time of strokeᵢ = mean(ts of all points in strokeᵢ)
tDiffList of intervals between adjacent taps = [ts₂−ts₁, ts₃−ts₂, …] (N−1 values, ms)
rTarget circle radius = circleL.r (= 56 in this set)
side(point)Per-point midpoint rule: if (x−cLx) ≥ (cRx−x) → right side, else left
dist(point)Distance from point to the "center of the chosen-side circle" = √((x−cx)²+(y−cy)²)
nL, nRNumber of strokes whose centroid is nearer the left / right circle (counted "per stroke")

10.1 Group A — tap count/size (from nᵢ)

FeatureFormulaRationale (why it separates PD)
Tap CountNTotal taps in the fixed time — fewer = slow tapping (bradykinesia)
Total PointsΣ nᵢTotal touch points — abnormally many = finger drag/tremor producing extra samples
Mean Tap Pointsmean(nᵢ)Mean points per tap — crisp taps (normal) give few points; PD holding/tremor gives more points per tap (each tap takes longer)
Max Tap Pointsmax(nᵢ)The "longest/most-tremulous" tap — captures the worst-case event
Std Tap Pointsstd(nᵢ)Spread of points per tap (raw scale)
tapPts_cvstd(nᵢ) / mean(nᵢ)Inconsistency of "tap size" normalized by the mean — removes the mean-points effect, leaving only "relative irregularity". PD taps short then long inconsistently → high value. (Better than Std Tap Points, which is still scale-bound.)

10.2 Group B — spatial accuracy (from dist, r)

FeatureFormulaRationale
mean_accmean( dist / r ) over all pointsMean distance to target divided by radius (normalizes circle size) — <1 = inside the circle; higher = farther from target (low accuracy) → PD
acc_stdstd( dist / r ) over all points"Inconsistency" of accuracy — normal subjects hit the target consistently (low std); PD hits then misses, distances scatter (high std). A different dimension than mean_acc: one is "average accuracy", the other is "how consistent the accuracy is"

10.3 Group C — left-right balance (from nL, nR)

FeatureFormulaRationale
ppLR_strokemin(nL, nR) / max(nL, nR)
(if denominator 0 → 0)		Assign a "side" to each tap from its centroid (1 tap = 1 vote, left or right), then compare the smaller/larger side ratio. Value = 1 means equal left-right taps (good balance = Control); near 0 means leaning to one side (PD using a non-dominant hand/side, or mis-tapping the wrong side). Counted per stroke to remove the confound of points-per-tap (unlike the old version counted per point, which was skewed by a tremor producing many points on one side)

10.4 Group D — rhythm/timing (from ts, tDiff)

FeatureFormulaRationale
tDiff_meanmean(tDiff)Mean inter-tap interval — larger = slower tapping
tDiff_medianmedian(tDiff)Median interval (more outlier-robust than mean)
tDiff_minmin(tDiff)Fastest achievable rhythm — Control often has a clear fast burst
tDiff_maxmax(tDiff)Longest pause/hesitation (freezing/hesitation) → PD
tDiff_stdstd(tDiff)Rhythm irregularity — tapping not in a steady beat = PD arrhythmia (the #1 feature by SHAP)
tDiff_cvstd(tDiff) / mean(tDiff)Normalized rhythm variability — separates "erratic rhythm" from "just slow" (slow-but-steady has low cv; erratic PD has high cv)
tDiff_slopeSlope a from line fit tDiff ≈ a·(index) + b (least-squares over indices 0,1,2,…)Whether inter-tap intervals keep "increasing" through the test. a > 0 = progressive slowing / decrement — a hallmark of PD bradykinesia under repetitive tapping; a ≈ 0 = steady. Needs ≥3 values to compute, else = 0
duration_s(max(ts) − min(ts)) / 1000Total time spent on the whole task (seconds)
tap_rateN / duration_sTap rate (taps/second) — low = bradykinesia
fatigue_ratiomean(tDiff 2nd half) / mean(tDiff 1st half)Compares second-half vs first-half rhythm — >1 = slowing over time (fatigue). Needs ≥4 tDiff values

10.5 Group E — tremor/path within each tap (intra-stroke geometry)

FeatureFormulaRationale
spread_meanMean over strokes of mean( √((x−mx)²+(y−my)²) ) (point distance to that tap's centroid)"Fuzziness" of points within each tap — crisp taps cluster tightly (low spread); tremor/drag scatters points (high spread) → tremor
spread_maxmax of the above spread over all strokesThe most tremulous/fuzzy tap
path_meanMean over strokes of Σ √((xⱼ−xⱼ₋₁)²+(yⱼ−yⱼ₋₁)²) (sum of consecutive point distances within a tap)Length of the line the finger drags within one tap — a single-point tap is short; drag/tremor makes the line long

10.6 Group F — app score

FeatureFormulaRationale
Scoredata['score']The score the app computes (shipped with the file) — high = performed well = tends to Control
